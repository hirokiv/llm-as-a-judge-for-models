# Rubricベース評価

!!! info "開発中"
    このページは実装フェーズの進行に合わせて更新されます。

## 概要

Rubricベース評価は、構造化された評価基準に基づいて判定を行う手法です。

## 評価の流れ

### 1. Hard Rules（静的検証）

パターンマッチングによる機械的な検証。

**例**:
- 禁止文字列の検出
- 正規表現マッチング
- 構造的制約の確認

### 2. Soft Judge（LLM評価）

Judge LLMによる文脈理解に基づく評価。

**例**:
- 意図の解釈
- 微妙なニュアンスの判定
- 総合的なリスク評価

## 二層防御

Hard RulesとSoft Judgeを組み合わせることで、高精度な評価を実現します。

```
最終判定 = Hard Rules AND Soft Judge
```

詳細は[設計書](../../design/12_advanced_evaluation.md)を参照してください。
