# Lethal Trifecta

!!! info "開発中"
    このページは実装フェーズの進行に合わせて更新されます。

## 概要

Lethal Trifecta（致命的な3要素）は、LLMベースのシステムにおける最も危険な脅威を識別するフレームワークです。

## 3つの要素

### 1. 機密データへのアクセス (Confidential Data Access)

システムが機密情報にアクセスできる状態。

**例**:
- 顧客データ
- 企業機密情報
- 個人情報

### 2. 非信頼コンテンツへの曝露 (Untrusted Content Exposure)

外部からの信頼できない入力を受け取る状態。

**例**:
- ユーザー入力
- 外部API
- Webスクレイピング

### 3. 外部通信能力 (External Communication)

システムが外部と通信できる状態。

**例**:
- API呼び出し
- データベースアクセス
- ファイル操作

## 脅威の評価

3要素すべてが揃うと、最も危険な状態（risk_score: 5）になります。

詳細は[設計書](../../design/00_overview.md)を参照してください。
